# Sidebar Navigation
A sidebar navigation component. The component contains the [`Sidebar`](#sidebar), [`Mobile`](#mobile),
[`MenuItem`](#menuitem) and [`SubmenuItem`](#submenuitem) sub-components and uses the [`NavigationProvider`](#navigationprovider) context for state management.

## Sub-components
The `SidebarNavigation` has the following sub-components [`Sidebar`](#sidebar), [`Mobile`](#mobile),
[`MenuItem`](#menuitem) and [`SubmenuItem`](#submenuitem)

---
### Sidebar
The sub-component `SidebarNavigation.Sidebar` is a `<nav>` wrapper component.

---
### Mobile
The sub-component `SidebarNavigation.Mobile` is a wrapper component over the
`MenuItem` sub-components for mobile view.

---
### MenuItem
The sub-component `SidebarNavigation.MenuItem` is a top-level menu item that is used
to group `SubmenuItem` sub-components.

---
### SubmenuItem
The `SidebarNavigation.SubmenuItem` renders the actual link to the desired page or destination.

---
## NavigationProvider
The `SidebarNavigation` component uses the `NavigationProvider` context to manage navigation state. This context provides the following properties:

- `activePath` - The current active path.
- `setActivePath` - Function to set the active path.
- `isMobileMenuOpen` - A boolean indicating if the mobile menu is open.
- `setMobileMenuOpen` - A function to set the mobile menu open state.

The `NavigationProvider` is shared between `SidebarNavigation` and `AppBarNavigation` components, allowing them to work together seamlessly.

## useNavigationContext
The `useNavigationContext` hook is used to access the `NavigationProvider` context.
The context provides the following properties:

- `activePath` - The current active path.
- `setActivePath` - Function to set the active path.
- `isMobileMenuOpen` - A boolean indicating if the mobile menu is open.
- `setMobileMenuOpen` - A function to set the mobile menu open state.
